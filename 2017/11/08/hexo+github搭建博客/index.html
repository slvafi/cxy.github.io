hexo+github搭建博客

发表于 2017-11-08 | 分类于 blog

前言

其实网上有很多的‘hexo+github搭建博客’的指南，只不过即使有指南，我把个人站换成个人博客的时候也还是遇到各种坑，所以决定自己写一篇踩坑指南，供大家借鉴。

初级

安装git

请到 git 官网下载安装。

安装node

请到 node 官网下载安装。

全局安装hexo

打开Git Bash，输入

npm install -g hexo

初始化hexo

新建一个文件夹，如“blog”，创建完毕后进入文件夹，打开Git Bash，依次输入

hexo init
hexo install

本地预览

到现在为止一个blog就搭建好了，然后在Git Bash中依次输入

hexo generate （可以简写成hexo g）
hexo server （可以简写成hexo s）

然后打开浏览器，在地址栏输入localhost:4000，就可以看到hexo给你准备的原始博客模板。

创建github仓库

在你的github上new一个Repository，起名叫xxx.github.io。

发布到github

在你的hexo文件夹下，找到_config.yml（以下称为站点配置文件）并打开，修改deploy项：

deploy:
    type: git
    repo: https://github.com/"你的github名字"/"刚刚起的仓库名".git
    branch: master

其中repo项可以直接复制

然后执行

hexo generate （可以简写成hexo g）
hexo deploy （可以简写成hexo d）

此时就已经发布到github上了，浏览器访问”刚刚起的仓库名”即xxx.github.io，就可以看到你的博客。

若报错类似未找到git，则输入

npm install hexo-deployer-git –save

再执行上述操作。

到现在为止，我们已经初步搭出了一个博客并发布到github，使其能外网访问，接下来我们将丰富我们的博客，并一一细数踩过的坑。

进阶

更改hellow world

从开始跟随我们的流程走下来之后，博客是搭建出来了，可是首页怪怪的，不太好看，第一篇博客居然是个hello world，浓浓的初级风格，太low，怎么办？

这里就要提一下了，你的博客必须至少有一篇，若你的博客里一篇博客都没有，hexo就会报出not found 404的错误，然后展示给我们一个错误的页面。

所以我们只能打开

/blog/source/_posts/hello world.md

把它改成你想要发布的新一篇博客的名字，比如“hexo+github搭建博客”。
然后用markdown的语法，在这个.md文件里写你想写的博客。

写完之后执行如下指令，注意，以后每次修改均需要执行如下指令：

hexo clean // 删除/blog/public文件夹
hexo g // 根据/blog/source重新编译/blog/public
hexo s // 本地预览
or
hexo d // 发布到github

现在，你的第一篇博客终于不是hellow world了！

配置博客信息

在站点配置文件中找到site并填写

# Site
title: 你的博客标题
subtitle: 副标题
description: 博客简介
author: 作者
avatar: 一个url，你的头像地址 // 可以自己添加
language: zh-Hans // 设置为简体中文
timezone:

发表一篇新博客

在你已经有了一篇博客的基础上，还想发表新的博客怎么办呢？

hexo new “你的博客名”

然后找到

/blog/source/_posts/“你的博客名”.md

来写你的博客。首先要配置.md文件头：

---
title: 你的博客名
date: 2017-11-08 15:52:57 // 自动生成的时间
categories: blog　// 分类
tags: [blog1,blog2] // 标签
---

修改主题

hexo的主题多种多样，这里我们以next为例。
在blog文件夹下打开Git Bash，输入

git clone https://github.com/iissnan/hexo-theme-next themes/next

然后打开站点配置文件，找到theme字段，并将其值更改为 next。

theme: next

选择Scheme

Scheme 是 next 提供的一种特性，借助于 Scheme，next 为你提供多种不同的外观。同时，几乎所有的配置都可以 在 Scheme 之间共用。
打开

/blog/themes/next/_config.yml

（以下称为主题配置文件）
搜索 scheme 关键字。 你会看到有几行 scheme 的配置，将你需用启用的 scheme 前面注释 # 去除即可。
别忘了在你不需要启用的 scheme 前面注释 #。

在菜单中添加分类和标签

在上面.md文件头，有两项categories和tags，在.md文件中加入这两项还不够，菜单还无法识别，需要我们通过一系列操作添加分类和标签。

首先，新建一个页面，命名为tags：

hexo new page "tags"

然后找到新建的.md文件，修改文件头为：

---
title: 标签名
date: 2017-11-08 15:52:57 // 自动生成的时间
type: "tags"　// 页面类型设置为tags
omments: false // 选填，是否开启留言功能，默认是开启disqus
---

然后在菜单中添加链接。编辑主题配置文件，添加 tags 到 menu 中，如下:

menu:
    home: / || home // 首页
    tags: /tags/ || tags // 标签
    categories: /categories/ || th // 分类
    archives: /archives/ || archive // 归档

你的menu项下可能有很多项，别忘了在你不需要启用的项前面注释 #。

添加readme

好了，我们的博客建设的差不多了，但还差一个关键的readme.md。打开

/blog/source

新建一个readme.md，然后打开站点配置文件，
编辑skip_render：

skip_render: readme.md

这样hexo引擎就不会把我们的readme.md渲染成html文件了。

添加自定义样式

如果你是个FE，你肯定不满足markdown的样式，在markdown里加标签、在标签中加行间样式又太蠢，怎么办呢？打开：

/blog/themes/next/source/css

你会看见一堆文件夹和一个main.styl，hexo的css都是基于stylus编译的，现在你可以自己创建一个文件夹，把你的css文件放进去，然后在main.styl里：

@import "你的文件夹/你的css.styl"

或者直接把你的样式加到

/_custom/custom.styl

图片相关

若你想把图片放在本地，则打开

/blog/source

新建一个存储图片的文件夹比如images，然后把你的图片放进去。
引用时

/images/图片.jpg
